# Consultatie bij Kennisplatform API's

Logius houdt een consultatie bij het Kennisplatform API's voor 2 beoogde nieuwe hoofdstukken voor API Design Rules (ADR) versie 2.2.
De consultatie loopt tot 2026-02-06.
Feedback en suggesties zijn welkom via api@logius.nl of via [issues](https://github.com/Logius-standaarden/API-Design-Rules/issues) op GitHub.
Help mee versie 2.2 klaar te maken ter vaststelling.

Bij het Technisch Overleg (TO) API wordt er gewerkt aan de nieuwe versie van de ADR, waarbij er regels zijn opgesteld voor het omgaan met tijd en foutafhandeling.
Dit naar aanleiding van een grote vraag van API experts voor standaardisatie op deze gebieden.
Deze regels zijn met experts totstandgekomen en het TO wil nu extra input van een breder publiek aan API experts die onderdeel zijn van het Kennisplatform API's.

Met deze openbare consultatie bieden wij belanghebbenden de gelegenheid om kennis te nemen van deze nieuwe hoofdstukken en te reageren op de bijbehorende nieuwe design rules.

## Hoofdstukken

Deze consultatie bevat een document die de twee nieuwe hoofdstukken gecombineerd bevat.
Beide hoofdstukken zijn nog in ontwerp binnnen het Technisch Overleg en dus nog geen formeel onderdeel van ADR 2.2.
Vanuit Logius vragen wij u om dit document te consulteren op de volgende twee hoofdstukken:
- [Date and time](./api-design-rules/#date-and-time)
- [Error Handling](./api-design-rules/#error-handling)

## Reageren?

Reageren kan via een issue of pull request op de [repository](https://github.com/Logius-standaarden/API-Design-Rules/issues), of per e-mail aan api@logius.nl.

Wij ontvangen graag inhoudelijke opmerkingen, vragen of suggesties uiterlijk 6 februari 2026.
